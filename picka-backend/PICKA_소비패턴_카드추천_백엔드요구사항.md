# PICKA — 소비패턴 기반 카드 추천 (백엔드 요구사항)

> **"내 카드 대신 쓸 때 혜택 많이 받는 카드 순위"** — 페르소나별 개인화 · 백엔드 설계 요청서

---

## ✅ 결론 — 백엔드 신규 기능입니다

현재 이 화면은 프론트에 박아둔 **정적 데이터**라 사용자가 달라도 같은 순위가 나옵니다.
진짜 개인화하려면 **사용자 결제 이력을 분석하는 백엔드 엔드포인트**가 필요하며, 지금은 존재하지 않습니다.

---

## 1. 지금 무슨 데이터를 쓰고 있나

추천 순위 화면은 프론트의 **정적 스냅샷**(`RECO_CARDS` · `CATEGORY_CARDS`)을 그대로 보여줍니다.
사용자 결제 패턴 반영은 **0%** 입니다.

- 결제 시점 **한 건 추천**(`/recommendations`)은 이미 있음
- 하지만 **"내 소비에 맞는 새 카드 추천"** 은 백엔드에 아직 없음

---

## 2. 왜 백엔드여야 하나

- **데이터가 백엔드에 있음** — 사용자 거래 전체 이력 + 카드 1,558장 × 혜택.
  프론트가 하려면 카드 DB 전체를 앱에 담아야 해서 불가능합니다.
- **기존 혜택 계산 엔진 재사용** — 월 한도·실적을 반영하는 `calculate_scored_benefit`를 그대로 씁니다.
- **이건 LLM 프롬프트가 아님** — 금액을 LLM에 맡기면 환각합니다.
  규칙 기반 스코어링으로 계산하고, LLM은 "왜 이 카드" 설명 문장에만 선택적으로 씁니다.

---

## 3. 추천 알고리즘 — 3단계

| 단계 | 내용 |
|:---:|---|
| **1. 소비 프로파일** | 유저 거래내역을 카테고리별로 합산해 **월평균 소비 벡터**를 만든다.<br>예) `user1 = {교육 118만, 쇼핑 29만, 카페 28만, 배달 23만 …}` |
| **2. 후보 카드 점수화** | 미보유 카드마다 예상 연혜택을 계산한다.<br>`예상 연혜택 = Σ_카테고리( 월소비[cat] × 혜택률[cat], 월한도 적용 ) × 12 − 연회비` |
| **3. 정렬 · 분리** | 예상 연혜택 내림차순 정렬 → 신용/체크로 나눠 각 **상위 3장** 반환. |

---

| 사용자 | 상위 소비 업종 | 맞는 추천 방향 |
|:---:|---|---|

## ⚠️ 선결 과제 — 카테고리 어휘 통일

거래내역의 업종 코드가 **뒤죽박죽**입니다. 이대로 계산하면 **대부분 0점**이 나옵니다.

- **거래내역**: 일부는 영어 코드 `TUITION` `MART` `DELIVERY`, 일부는 한글 `문구`
- **혜택 기준**: 한글 `마트/쇼핑` `배달앱` `카페/디저트`

→ **카테고리 정규화 매핑**(`DELIVERY→배달앱`, `MART→마트/쇼핑`, `TUITION→교육/육아` …)이
스코어링 **이전에** 반드시 필요합니다.
(`merchant_aliases`처럼 매핑 테이블/딕셔너리 한 벌)

## 5. 엔드포인트 스펙 — 백엔드에 그대로 전달

### 신규 API — 소비패턴 기반 카드 추천

```
GET /api/v1/users/{user_id}/card-recommendations?type=credit|check&limit=3
```

**동작**

1. 해당 유저 거래내역을 카테고리별로 합산해 월평균 소비 벡터 생성 (기간: 최근 3개월 등)
2. 거래 업종(영어/한글 혼재)을 혜택 기준(한글)으로 **정규화** ← 매핑 필요
3. 미보유 카드마다 소비 벡터 × 혜택률(월 한도 적용)로 예상 연혜택 계산 — 기존 엔진 재사용
4. `type`(신용/체크) 필터 → 예상 연혜택 내림차순 → 상위 `limit`개 반환

**응답** _(프론트가 이미 쓰는 필드 그대로)_

```json
// 예상 혜택 금액은 규칙 기반으로 계산할 것. LLM 으로 금액 생성 금지(환각).
{
  "cards": [
    {
      "id": 53,
      "name": "카드 상품명",
      "issuer": "카드사",
      "benefitName": "이 유저 소비 기준 최대 혜택 항목",
      "rate": 5.0,
      "total": 480000,      // 예상 연혜택(원)
      "fee": 15000,         // 연회비
      "url": "카드고릴라 링크",
      "image_url": "카드 이미지"
    }
  ]
}
```
