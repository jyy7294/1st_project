## 1. cards_master_final.csv (1,558행)

### 1-1. 원본 유지 컬럼

| 컬럼 | 타입 | 설명 |
|---|---|---|
| 카드번호 | int | **PK.** 카드고릴라 카드 고유 ID |
| 구분 | str | `신용카드` / `체크카드` |
| 카드사 | str | 발급사명 (54개사) |
| 카드명 | str | 상품명 (동명 카드 존재 — PK로 쓰지 말 것) |
| 브랜드 | str | 원본 브랜드 표기. 결측 대체값 `국내전용`, `미상` 포함 |
| 전월실적 | int | 카드 공통 전월실적 조건(원). 0 = 조건 없음 |
| 연회비 / 주요혜택 / 전체상세혜택 / 이벤트혜택 | str | 원문 보존용. 계산에 사용 금지 (파싱 완료된 컬럼 사용) |
| 카드이미지 / 상세URL | str | CDN 이미지, 카드고릴라 상세 페이지 |

### 1-2. 파생 컬럼

| 컬럼 | 타입 | 설명 / 결측 규칙 |
|---|---|---|
| 연회비_국내 | float | 국내전용 연회비(원). **NULL = 해외전용 카드 등 국내 옵션 없음** |
| 연회비_해외 | float | 해외겸용 연회비(원). **NULL = 국내전용 카드** |
| 연회비_최소 | float | min(국내, 해외). 비용 계산 기본값. 0 = 무료 |
| 연회비_무료 | bool | 연회비_최소 == 0 |
| 브랜드_목록 | str | `\|` 구분 정규화 목록 (빈 문자열 = 국제 브랜드 없음) |
| 브랜드_수 | int | 선택 가능 브랜드 수 (0~5) |
| 브랜드_VISA … 브랜드_Diners_Club | bool | 8개 브랜드 원핫 |
| 국내전용카드 | bool | 국제 브랜드 없음 (130건) |
| 해외결제가능 | bool | 국제 브랜드 1개 이상 (BC 단독은 제외, BC Global은 포함) |
| 혜택수 / 혜택카테고리수 | float | 유의사항 제외 혜택 블록 수 / 고유 카테고리 수 |
| 옵션형혜택보유 | bool | 택1 옵션그룹 보유 여부 |
| 월최대혜택합계 | float | 한도 확보 혜택의 월최대혜택액 합 (참고용) |
| 최대할인율 | float | % 단위 혜택 중 최대값 |
| 혜택율_{카테고리} ×29 | float | 카테고리별 최대 %혜택 룩업 매트릭스. NULL = 해당 카테고리 %혜택 없음 |
| 통합한도_월 | float | 카드 전체 통합 월 할인한도(원). 유의사항 역참조 추출 (243건). NULL = 미확인 |
| 한도파악률 | float | 0~1. 이 카드 %혜택 중 한도 정보(수치 또는 '한도없음')가 확보된 비율. **추천 신뢰도 지표** |
| 스냅샷일자 | str | 데이터 수집 기준일 (YYYY-MM-DD) |

## 2. card_benefits_final.csv (6,981행)

| 컬럼 | 타입 | 설명 / 결측 규칙 |
|---|---|---|
| 혜택ID | str | **PK.** `{카드번호}-{순번2자리}` |
| 카드번호 | int | FK → cards_master |
| 혜택순번 | float | 카드 내 원문 순서. 소수점(x.01~) = 인라인 옵션 분해로 생성된 파생 행 |
| 카테고리 | str | 29개 표준 카테고리 + `구독/멤버십`, `유의사항` 없음(혜택만 수록) |
| 카테고리목록 | str | 다중 카테고리 혜택의 `\|` 구분 목록. NULL = 단일 카테고리(카테고리 컬럼 사용) |
| 카테고리_원본 | str | 크롤링 원본 카테고리 라벨 |
| 혜택유형 | str | 할인 / 포인트 적립 / 캐시백 / 마일리지 적립 / 무료제공 / 면제·우대 / 무이자할부 / 바우처 / 기타 |
| 혜택값 | float | 혜택 크기. NULL = 정량 표현 없는 혜택(라운지 무료 등) — 스코어링 제외 |
| 혜택단위 | str | `%` / `원` / `원/L` / `마일/천원` |
| 실적조건 | float | 이 혜택의 전월실적 조건(원). NULL = 블록 내 명시 없음 → 카드 공통 `전월실적`만 적용 |
| 한도_회당 / 한도_월 / 한도_일 / 한도_연 | float | 각 주기별 한도(원). 원문에 명시된 것만 수록 |
| 한도_월_최대구간 | float | 실적 구간표가 있는 경우 최상위 구간 한도 (한도_월은 최소 구간 = 보수적) |
| 횟수_일 / 횟수_월 | float | 적용 횟수 제한 |
| 월최대혜택액 | float | **스코어링용 통합 캡.** min(월한도, 일×30, 연/12, 회당×횟수). NULL 처리 규칙은 README 참조 |
| 한도없음 | bool | 원문에 "한도 없이/무제한" 명시 (181건) — 캡 미적용 대상 |
| 한도확보 | bool | 이 혜택의 한도 정보 확보 여부 (월최대혜택액 있음 OR 한도없음 OR 카드 통합한도 있음) |
| 적용범위 | str | `일반` / `정기결제한정`(구독료 결제 건만) / `연1회기프트`(월 스코어링 제외) |
| 옵션그룹 | float | 택1 그룹 ID. 같은 값끼리 합산 금지, max 1개 반영. NULL = 일반 혜택 |
| 옵션헤더 | bool | 옵션을 나열만 하는 안내 블록 — 스코어링 제외 |
| 옵션형 | bool | 옵션 관련 블록 플래그 (그룹 미확정 포함) |
| 가맹점수준 | str | `업종전체` / `복수가맹점` / `단일가맹점` / `불명` — 커버리지 계수 매핑용 |
| 가맹점목록 | str | `\|` 구분 가맹점명. 결제 시점 직접 매칭 키. NULL = 업종 전체 대상 |
| 요약 / 상세 | str | 원문 (상세는 CSV에만 존재, 앱 상세화면 표시용) |
| 스냅샷일자 | str | 수집 기준일 |

## 4. card_database.json

`meta`(스냅샷일자·건수·계산 가정) + `cards[]`. 각 카드 객체는 마스터 주요 필드
(혜택율 매트릭스·원문 컬럼 제외) + `혜택[]` 배열(혜택 테이블에서 `상세` 제외 전 컬럼).
